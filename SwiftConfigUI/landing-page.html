Getting Started with Ada

Setting up Ada on your macOS system is straightforward. Follow these steps to install and configure your AI assistant:

Clone the Repository

Open Terminal and run the following command to clone the repository:

git clone https://github.com/yourusername/poc-realtime-ai-assistant.git

Navigate to the project directory:

cd poc-realtime-ai-assistant

Install Dependencies

Install UV, the modern Python package manager:

curl -sSf https://install.astral.sh/uv | bash

Install project dependencies:

uv sync

Configure the Assistant

Open the SwiftUI configuration app in Xcode:

open SwiftConfigUI/AIAssistantConfig.xcodeproj

Build and run the app (⌘R). Use the interface to:

  • Enter your OpenAI API key
  • Customize the assistant's name and behavior
  • Configure file paths and database settings

Launch Ada

You can launch Ada in two ways:

  1. Using the Swift UI: Navigate to the "Launch" tab and click "Launch Assistant"
  2. Using Terminal: Run uv run main or with initial prompts: uv run main --prompts "Hello, Ada!"

Start Interacting

Once launched, you can interact with Ada through:

  • Voice commands: Speak naturally, prefacing requests with "Hey Ada" if desired
  • Text input: Type commands directly in the terminal window

Troubleshooting

Microphone Not Working

If Ada can't hear you:

  1. Check macOS microphone permissions in System Preferences > Security & Privacy > Microphone
  2. Ensure your microphone is selected as the default input device
  3. Restart the assistant after making changes
API Key Issues

If you're encountering API key errors:

  1. Verify your OpenAI API key in the Swift configuration app
  2. Ensure your OpenAI account has access to the Realtime API
  3. Check for any billing issues in your OpenAI account
Database Connection Problems

If database operations aren't working:

  1. Verify the database paths in your configuration
  2. For SQLite/DuckDB: Reset using provided scripts in README
  3. For PostgreSQL: Check connection string and credentials
Audio Response Issues

If audio responses are cutting out or not playing:

  1. Check your output device settings
  2. This is a known issue being addressed in future updates
  3. As a workaround, restart the assistant

def run_shortcut(shortcut_name, parameters=None):
    import subprocess
    
    # Build the URL scheme to call the shortcut
    url = f"shortcuts://run-shortcut?name={shortcut_name}"
    if parameters:
        # Add parameters as needed
        for key, value in parameters.items():

More Advanced: AppleScript Integration

For deeper system integration, AdA can use AppleScript to control applications:

# Running AppleScript from Python
def run_applescript(script_text):
    import subprocess
    cmd = ["osascript", "-e", script_text]
    result = subprocess.run(cmd, capture_output=True, text=True)
    return result.stdout.strip()

# Example: Ada controlling mail app

Ada Intelligence: Current State and Future Evolution

While Ada already offers impressive capabilities, it's important to understand her current limitations and the expected timeline for evolution into a more comprehensive assistant.

Current Limitations

Limited System Awareness

Ada currently lacks real-time awareness of your desktop state, running applications, and screen content without explicit integration.

Permission Constraints

macOS security model limits Ada's ability to control system functions without explicit user authorization for each domain.

Integration Gaps

Ada requires manual setup to connect with third-party applications and services, without native integration capabilities.

Context Retention

Ada has limited ability to maintain complex contextual understanding across sessions without explicit memory management.

Proactive Assistance

Ada primarily operates in reactive mode, responding to commands rather than proactively offering assistance based on context.

Offline Capabilities

Ada requires internet connectivity for core functions, limiting usability in offline environments or low-connectivity situations.
